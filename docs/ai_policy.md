# AI Policy

AI coding assistants (ChatGPT, Claude, Copilot, and similar) are **permitted** in this course
and are treated as a normal part of a modern analyst's workflow. The course position is:

> **Use AI, then verify.** You are responsible for understanding, checking, and standing
> behind anything an assistant produces on your behalf.

## What this means in practice

- **Understand before you submit.** If you cannot explain what a chart shows or why a line of
  code works, it is not ready to hand in.
- **Verify every AI-generated chart.** Assistants confidently produce misleading or incorrect
  visualisations. Each completed notebook includes a *verification checklist* — apply it.
- **Cite assistance.** When AI materially shaped your project work, note it briefly (which
  tool, for what). This mirrors how professionals disclose tooling.
- **Data stays synthetic / non-confidential.** Do not paste real personal or confidential data
  into external AI tools.

## Verification checklist for AI-generated charts and code

1. Does the chart answer the actual question asked?
2. Are the axes, units, and scales correct and honest (no truncated/ dual axes by accident)?
3. Is the chart type appropriate for the data (e.g. not a pie chart for many categories)?
4. Does the code run top-to-bottom on the real dataset, not a hallucinated one?
5. Are column names, aggregations, and filters what you intended?
6. Did it silently drop, duplicate, or mis-join rows?
7. Could a reader be misled by colour, ordering, or missing context?

If you cannot answer all seven, fix it before relying on the output.

## On repository creation

The prototype structure and content of this repository were drafted with AI assistance and
